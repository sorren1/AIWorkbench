# Threat model

## Scope

This model covers the functional browser-local authorization demonstration and gitignored local sandbox CLI. External providers and enterprise integrations remain simulated. It does not claim that local storage or a local filesystem resists a malicious machine owner.

| Threat                            | Local mitigation                                                                                                                                                  | Residual / production requirement                                                                              |
| --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| Unauthorized approval             | Closed synthetic persona IDs, persona/scope checks, reason requirement, fail-closed policy lookup                                                                 | Authenticate humans/workloads and authorize server-side                                                        |
| Self-approval / separation bypass | Requester binding, self-approval prohibition, non-overlapping reviewer/administrator scopes                                                                       | Use authoritative organization/team and conflict data                                                          |
| Approval replay                   | Bind arguments, versions/hashes, target/context digests, expiry/cache TTL; completed runs cannot resume                                                           | Transactional single-use consumption and idempotency keys                                                      |
| Stale decision                    | Recompute all bindings and append `INVALIDATED` on mismatch                                                                                                       | Trusted context/version services and compare-and-swap                                                          |
| Confused deputy                   | Intersect human, delegation, stage, agent, tool, and resource authority; discovery grants nothing                                                                 | Audience/resource-bound tokens and workload attestation                                                        |
| Delegation escalation             | Deny delegated scopes absent from the initiating persona                                                                                                          | Signed delegation/token-exchange claims and revocation                                                         |
| Policy downgrade/tamper           | Schema validation, content hashes, exact policy/version binding                                                                                                   | Signed bundles, trusted distribution, rollback protection                                                      |
| Path escape                       | Host glob intersection plus server real-path/symlink checks and bounded writes                                                                                    | OS/container isolation, read-only mounts, egress controls                                                      |
| Network exfiltration              | Deny network-enabled sandbox requests; no network tool exists                                                                                                     | Enforced egress and credential isolation                                                                       |
| Event-log mutation                | Hash-chained events expose accidental/local mutation                                                                                                              | Append-only/WORM storage, signatures, trusted timestamps                                                       |
| Browser impersonation             | Persona switching is labeled synthetic/local                                                                                                                      | Derive identity from an authenticated session                                                                  |
| Prompt injection in context       | Treat records as data; enforce source/type/scope rules; revoke the checked-in attack fixture before selection; tools remain separately authorized                 | Content sanitization, trust scoring, model-side defenses, and monitored incident response                      |
| Stale or revoked context          | Evaluate explicit state and effective TTL at assembly and resume; mismatch invalidates the bound pack                                                             | Authoritative source change events, trusted clocks, and transactional invalidation                             |
| Poisoned memory / prior-run data  | No autonomous learning; versioned hashed records; prior failure/fix records require an explicit episodic policy                                                   | Moderated ingestion, signed provenance, quarantine, and rollback across shared stores                          |
| Over-broad context / data leakage | Stage/persona/agent/scope/sensitivity allow-lists plus byte/token budgets and visible exclusions                                                                  | Tenant isolation, DLP, row/object authorization, and auditable retrieval infrastructure                        |
| Trace leakage                     | Allow-listed metadata namespace, bounded values, credential/multiline redaction, no prompt/source/output/arguments, fixed error categories                        | Central telemetry access control, encryption, retention, regional policy, and DLP                              |
| Budget bypass                     | Content-hashed policy, pre-operation exact-count checks, measured-time checks, trace events, failure evidence, non-zero stop exit                                 | Durable distributed counters, reservation/settlement, queue cancellation, and billing guard                    |
| Arbitrary public code execution   | Static site has no runner endpoint; local CLI accepts no repository, patch, command, or visitor input                                                             | Authenticated hardened worker service if live execution is ever separately authorized                          |
| Sandbox path/symlink escape       | Exact allow list, segment and real-path checks, all-symlink rejection, unexpected-file/change verification, disposable workspace                                  | Stronger OS primitives and isolation against a malicious local account / TOCTOU races                          |
| Sandbox resource abuse            | Fixed commands, network none, non-root user, read-only mounts/root, dropped capabilities, CPU/memory/PID/time/output limits                                       | Dedicated workers, quotas, admission control, kernel hardening, and abuse monitoring                           |
| Evidence tampering                | Versioned schema, embedded content/output/tree hashes, context binding, canonical pack digest, index/Markdown binding                                             | Signatures, trusted timestamps, append-only retention, and independent attestation                             |
| Container/image compromise        | Digest-pinned Node base; minimal local image removes unused npm; exact built image ID is Trivy-gated; no Docker socket, secrets, home, or project-root mount      | Signature verification, patched dedicated hosts, and runtime defense                                           |
| Dependency compromise             | Exact npm lockfile, clean install, high/critical audit gate, pull-request dependency review, CycloneDX inventory, and license policy                              | Registry compromise, maintainer takeover, and advisories published after validation remain possible            |
| Secret retained in source/history | Redacted Gitleaks scan covers the tracked worktree snapshot and reachable Git history; existing output-pattern scan remains defense in depth                      | Novel or encoded credentials may evade signatures; any real exposed credential must still be rotated           |
| Scanner or CI compromise          | Scanner containers and GitHub Actions are immutable-digest/SHA pinned; versions, target IDs, report hashes, and advisory-database metadata are recorded           | Artifact signatures and provenance are not independently verified; the runner and Docker daemon remain trusted |
| Report/SBOM tampering             | Detailed artifacts are hash-bound into a sanitized source-linked summary and retained as CI artifacts; SARIF source snippets are removed                          | Reports are not signed attestations, transparency-logged evidence, or immutable long-term storage              |
| Stale suppression                 | Central exceptions require scanner, rule, exact path, reason, reviewer, review date, and expiry; malformed, duplicate, unmatched, or expired entries fail         | Reviewer identity is repository metadata rather than an authenticated approval                                 |
| Gateway master-key exposure       | Master key exists only in local process/Compose environment; browser rejects gateway access; adapter accepts loopback HTTP only; safe errors omit response bodies | OS secret store, workload identity, rotation, protected administrator API, and host hardening                  |
| Scoped credential leakage         | Opaque lease contract; raw key only in owner-oriented gitignored recovery state; redaction tests cover traces/evidence/build; alias contains no personal data     | Encrypted credential store, memory hardening, audit alerts, and short-lived token exchange                     |
| Fallback drift                    | Registry-hashed ordered aliases; catalog preflight; internal gateway retries disabled; each attempt/event traced                                                  | Signed centralized routing policy, provider health governance, and change approval                             |
| Cost runaway                      | Virtual-key spend ceiling plus local token/cost/time/call budgets; stop events fail the run; 15-minute lease                                                      | Durable reservation/settlement, billing-webhook reconciliation, organization quotas, and kill switches         |
| Provider data retention           | Fixed synthetic live-test request; prompts/responses excluded from evidence; live profile is explicit and provider identity is recorded                           | Provider contract, zero-retention controls where available, residency/DLP review, and deletion process         |

Tests exercise unauthorized personas, self-approval, expiry, context TTL/revocation/scope/budget rules, poisoned-record exclusion, replay, stale arguments and context, version changes, path escape, symlink escape, allow-list enforcement, unexpected files, timeout, deterministic output normalization, trace hierarchy/redaction/binding, budget warnings/stops/repair limits, exact-versus-estimated accounting, failed/successful evidence, evidence tamper detection, deny precedence, malformed schemas, process cleanup, event-chain verification, sandbox context mismatch rejection, gateway alias reconciliation, model allow lists, explicit fallback, scoped-key revocation, interrupted cleanup, and secret exclusion.

The repository-wide supply-chain boundary is summarized in `THREAT_MODEL.md`, with reproducible artifacts and enforcement described in `docs/release-evidence.md`.
The Docker-specific trust boundary and residual risks are detailed in `docs/sandbox-security-model.md`.
The optional provider path and its different network boundary are detailed in `docs/model-gateway-and-routing.md` and `docs/adr/local-model-gateway.md`.
