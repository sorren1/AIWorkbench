A governed, human-in-the-loop control plane for AI-assisted software delivery.

Engineering thesis

Generation is the easy part. Control is the product.

A credible coding-agent system must explain what was authorized, which context was selected, where tools may act, when a human must decide, and which evidence supports the result.

01

Authorization

Stage-scoped capabilities and lifecycle gates should define what may run.

02

Context

Explicit manifests can make provenance, freshness, exclusions, and digests reviewable.

03

Tool boundaries

Read, bounded-write, and privileged operations require distinct policy treatment.

04

Human approval

Risky actions should stop at an explicit human decision point.

05

Evidence

Artifacts, review state, validation, budgets, and trace data should travel with the change.

Architecture

Separate steering from execution.

Four responsibility planes keep local product behavior, simulated adapters, and future production infrastructure conceptually distinct.

Control plane

Workflow state, registry policy, approvals, budgets, and audit references.

Execution plane

Isolated workspaces, stage runtimes, model adapters, and bounded tool calls.

Context plane

Issue, repository, documentation, schema, and prior-decision context packs.

Validation plane

Acceptance coverage, review decisions, evidence envelopes, and release gates.

Illustrative adapter set

Jira, GitHub, Angular, .NET, and Oracle provide one technically specific reference stack behind vendor-neutral issue, repository, UI, service, and database boundaries. They are synthetic configuration examples, not a prescribed environment.

Deliberate tradeoffs

Small enough to inspect. Explicit about what comes next.

Static by design

A Vite multi-page build keeps hosting simple and the public narrative readable without JavaScript.

Deterministic by design

Fixtures make state transitions reproducible, but do not stand in for integration or model reliability.

Focused by design

The registry and policy model govern delivery-stage agents rather than becoming a generic agent catalog.

Production boundary

Identity, secrets, isolated execution, durable storage, telemetry collection, and provider adapters remain future infrastructure.
